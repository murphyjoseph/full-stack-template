# @repo/ui

Park UI (Ark UI) component library with Panda CSS. 62 pre-styled components, per-component exports for tree-shaking.

**Why:** Single source of truth for the design system. Components are source-compiled by Next.js via `transpilePackages` — no separate build step for `.tsx` files.

## Usage

```tsx
import { Button } from "@repo/ui/button";
import { Heading } from "@repo/ui/heading";
import { Container, Stack } from "@repo/ui/jsx";
import * as Card from "@repo/ui/card";
import { css } from "@repo/ui/css";
import { createAppConfig } from "@repo/ui/panda-config";
```

Always import per-component (`@repo/ui/button`), never from a barrel.

## Structure

```
packages/ui/
  src/components/    # 62 Park UI components (.tsx, source-compiled)
  preset/            # Shared design system preset (tokens, recipes)
  recipes/           # Custom recipes (group, breadcrumb, scroll-area, etc.)
  lib/               # createStyleContext (injected into styled-system by codegen hook)
  styled-system/     # Generated by panda codegen (not committed)
```

## Commands

```sh
pnpm --filter @repo/ui build   # panda codegen (generates styled-system/)
pnpm --filter @repo/ui dev     # panda codegen --watch
```

**When to run watch mode:**
- Modifying recipes (`packages/ui/recipes/`)
- Changing design tokens (`packages/ui/preset/`)
- Adding new components (triggers codegen hook)

**Not needed when:**
- Only editing component `.tsx` files (Next.js hot-reloads via transpilePackages)

## Config Exports

Consuming apps can import shared configs:
- `@repo/ui/panda-config` — `createAppConfig()` helper that extends the UI preset
- `@repo/ui/postcss-config` — Standard PostCSS config for Panda CSS
